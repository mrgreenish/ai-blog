OpenAI's current Voice settings split the experience three ways: **Live** is the newest, simultaneous listen-and-speak option; **Advanced** is the earlier real-time mode with supported mobile features such as video and screen sharing; **Standard** waits for a turn. Availability varies by plan, region, workspace settings, and app version. [OpenAI's Voice guide](https://help.openai.com/en/articles/20001274) is the source of truth for the switches and limits.

For Codex, the useful change is conversational coordination rather than voice-controlled coding. In the ChatGPT desktop app on macOS or Windows, eligible users can select Codex, start Voice, and ask it to begin a task, check progress, or coordinate agent work — while interrupting naturally and reading the live text in the chat. Voice keeps the tools and permissions of the selected Codex experience; it is not a standalone Codex mode on web or mobile, though paired iOS remote access is supported. [OpenAI documents the desktop flow here](https://help.openai.com/en/articles/20001275-chatgpt-work-and-codex).
